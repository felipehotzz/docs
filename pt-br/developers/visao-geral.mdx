A **Loop Developer Platform** é a área técnica do Loop para times de TI configurarem integrações corporativas sem entrar no fluxo operacional do aplicativo.

Ela foi desenhada para implantação, homologação e auditoria. O objetivo é responder com clareza:

- como o Loop acessa um sistema externo;
- quais campos serão consumidos;
- o que será criado, atualizado ou inativado;
- quais regras protegem a base de colaboradores;
- quando uma execução aconteceu e qual foi o resultado.

<Note>
  A Developer Platform não é uma API pública aberta. Ela é uma área de implantação e governança para integrações contratadas com o Loop.
</Note>

## Primeiro conector: Senior HCM

A primeira integração suportada pela Developer Platform é com a **Senior HCM**.

O conector Senior tem como foco sincronizar a base de colaboradores do cliente com o Loop:

- novos colaboradores;
- alterações cadastrais;
- áreas, cargos e centros de custo;
- status ativo/inativo;
- desligamentos;
- dados adicionais para segmentação.

## Fluxo de implantação

1. TI informa endpoint, ambiente e modelo de autenticação.
2. Loop configura a conexão em homologação.
3. TI e Loop ajustam o mapeamento de campos.
4. A plataforma gera um preview sem alterar colaboradores.
5. TI valida os impactos: criados, atualizados, inativados e preservados.
6. Após homologação, a integração pode ser promovida para produção.
