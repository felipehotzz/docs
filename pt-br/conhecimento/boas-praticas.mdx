---
title: "Boas práticas"
description: "Como escrever páginas úteis para pessoas, busca e IA"
---

Boas páginas reduzem dúvidas repetidas, melhoram a busca e deixam a IA mais confiável. O objetivo é escrever para pessoas, mas com estrutura suficiente para o Loop entender o conteúdo.

## Estrutura recomendada

Use este formato como ponto de partida:

1. Título claro.
2. Resumo curto do que a página resolve.
3. Quando usar ou quando a regra se aplica.
4. Passo a passo ou regras principais.
5. Exemplos.
6. Perguntas frequentes.
7. Responsável pela informação.
8. Data de última revisão, quando fizer sentido.

## Escreva para busca

- Use os termos que colaboradores realmente pesquisam.
- Inclua sinônimos importantes.
- Explique siglas na primeira ocorrência.
- Prefira títulos específicos.
- Separe assuntos diferentes em páginas diferentes.

## Escreva para IA

A IA responde melhor quando a página tem contexto explícito.

Bom:

> Para solicitar reembolso, envie a nota fiscal pelo sistema financeiro até o quinto dia útil do mês seguinte.

Ruim:

> Fazer pelo sistema até a data combinada.

O segundo exemplo depende de contexto externo e tende a gerar respostas piores.

## Evite duplicidade

Se duas páginas falam sobre a mesma regra com versões diferentes, a busca pode encontrar ambas. Isso confunde pessoas e IA.

Antes de criar uma nova página, pesquise se já existe uma página parecida. Se existir, prefira atualizar a fonte principal.

## Mantenha conteúdo vivo

Revise páginas quando:

- uma política mudar;
- um benefício for atualizado;
- um processo trocar de sistema;
- uma campanha terminar;
- a busca começar a retornar respostas ruins;
- colaboradores continuarem perguntando algo que deveria estar claro.

## Conteúdo sensível

Para temas sensíveis, mantenha linguagem objetiva e fonte oficial. Evite interpretações soltas em páginas de política, jurídico, compliance, segurança ou benefícios.

<Note>
  A melhor página para IA é a mesma melhor página para uma pessoa: clara, atual, específica e fácil de verificar.
</Note>
